1. Problem Statement

Most people wear only 20% of their wardrobe.

Not because they lack clothes — because they can't see what they own, can't match what they have, and can't remember what they've worn. The average person spends 17 minutes every morning deciding what to wear, often defaulting to the same familiar 20% while the other 80% hangs untouched.

Existing solutions fail in five critical ways:

PrivacyUpload photos to cloud serversAll data stays on device
IntelligenceBasic tagging, no vision AIGemma 4 multimodal analysis
CostSubscription models ($5–20/month)Free, runs on your hardware
Cultural awarenessWestern-centric only20+ global garment types
PlatformWeb onlyWeb + Native Android APK

Wearly solves all five gaps with a single local-first AI application powered entirely by Gemma 4.

2. Solution Overview

Wearly is a local-first AI wardrobe assistant that uses Gemma 4 as its central intelligence engine for:

  • Computer vision — photographing and identifying garments with 5-step structured analysis
  • Outfit generation — creating complete outfits from your actual wardrobe
  • Natural language styling — conversational AI stylist with deep fashion knowledge
  • Self-modification — writing and deploying new features from plain English descriptions
  • Voice interaction — speaking results aloud with on-device Web Speech API

The system runs entirely on your own machine via Ollama (zero data leaves your device) with a transparent cloud fallback to Groq when Ollama isn't running.

Why Gemma 4 Specifically

  • Multimodal by default — vision + language in a single model call
  • Efficient enough to run locally — no GPU cluster required, works on a MacBook
  • Strong instruction following — reliably produces structured JSON output from fashion prompts
  • Open weights — aligns with Wearly's privacy-first philosophy

3. Gemma 4 Integration — The Core Intelligence

3.1 Unified AI Client

All Gemma 4 calls go through a single abstraction layer in src/lib/ai.ts. The backend is detected on every request with a 2-second timeout — transparent to all feature code:

async function detectBackend(): Promise<'ollama' | 'groq' | null> {
  try {
    const res = await fetch('http://localhost:11434/api/tags',
      { signal: AbortSignal.timeout(2000) });
    if (res.ok) return 'ollama';  // Gemma 4 local — private, free
  } catch {}
  if (process.env.GROQ_API_KEY) return 'groq'; // Cloud fallback
  return null;
}

3.2 Vision Analysis — 5-Step Fashion Intelligence

When a user photographs a garment, Gemma 4 runs a structured 5-step reasoning chain:

  1. Identify the item — isolate from background, classify category
  2. Detect the pattern — solid, plaid, stripes, check, floral, graphic, houndstooth, tie-dye, brocade, batik, paisley
  3. Recognise the material — from visual cues: flannel (brushed matte), denim (diagonal twill), silk (high sheen, fluid drape), velvet (pile texture), linen (visible weave, slight wrinkle), boucle (loopy), chiffon (sheer)
  4. Analyse colour precisely — 60+ named shades. Critical rule: for plaid/stripes, the background field colour wins, not the grid lines
  5. Construct the name — formula: [Color] [Pattern] [Material] [Category]

Example outputs: "Dark Green Plaid Flannel Shirt", "Ivory Silk Midi Dress", "Slate Blue Linen Trousers"

3.4 Smart Mirror — Real-Time Gemma Vision

Mount your Android phone inside your wardrobe. Open the Mirror. Gemma 4 watches via the camera in real time. When you hold up a garment:

  1. Identifies the item without requiring a photo tap
  2. Searches your localStorage wardrobe for matching complementary pieces
  3. Builds a complete outfit: top + bottom + shoes + accessories
  4. Speaks the suggestion aloud: "Try this with your Navy Slim Chinos and White Leather Sneakers"
  5. Detects uncatalogued items: "I see a navy blazer not in your wardrobe yet — add it?"

Wake Lock API keeps the screen on so you never touch the phone while dressing.

3.5 Self-Evolving Code Generation

The most experimental feature: describe a feature in plain English, Gemma 4 writes it. The system reads the codebase, produces a complete implementation (component + API route + types), creates a GitHub Pull Request, and shows a live animated pipeline: Received → Building → Deployed. The app ships new features to itself using Gemma 4 as the developer.

4. Key Features Deep Dive

4.1 Wardrobe Catalogue

The core catalogue stores every garment as a structured object with all vision-extracted fields plus user metadata (wear count, last worn date, purchase price, CO₂ estimate). Features: grid/list views, multi-filter, natural language search, wear tracking.

4.2 Outfit Builder — Three Modes

  • AI-Generated — Gemma 4 reads your wardrobe JSON and applies colour theory (60/30/10 rule), occasion logic, seasonal fit, and personal style profile
  • Smart Mirror — real-time camera-based suggestions as you hold items up
  • Manual — drag-and-drop builder with AI validation (Gemma 4 rates the combination)

4.3 Voice-First Interaction

Every action speaks aloud using Web Speech API — fully on-device. Technical challenge: desktop browsers require a synchronous user gesture before speechSynthesis fires, but AI responses arrive asynchronously. Solved with a two-phase unlock system:

// Phase 1: capture-phase listener on first user interaction
document.addEventListener('click', unlock, { capture: true });

function unlock() {
  const silent = new SpeechSynthesisUtterance('​'); // zero-width space
  silent.volume = 0;
  window.speechSynthesis.speak(silent); // unlocks API for entire session
  _unlocked = true;
  if (_pending) setTimeout(() => _doSpeak(_pending.text, _pending.rate), 80);
}

// Phase 2: speak() queues before unlock, plays immediately after
export function speak(text: string, rate = 0.93) {
  if (!_unlocked) { _pending = { text, rate }; return; }
  _doSpeak(text, rate);
}

4.4 Trip Planner

Input: destination, duration, activities. Gemma 4 reads your actual wardrobe and generates a packing list using only items you already own — applying weather, activity filtering, mix-and-match maximisation, and capsule wardrobe principles.

4.5 Sustainability Tracker

  • Cost-per-wear — purchase price ÷ wear count, updated in real time
  • CO₂ estimation by fabric type (synthetic vs natural vs pre-owned)
  • 60-day idle detection — flags unworn items, suggests donate/sell/re-discover
  • Wardrobe score (0–100) combining diversity, wear distribution, eco profile
  • Circular economy — one-click marketplace listing generation (Carousell, eBay, Depop)

7. Android App — Native TWA

Technology: Trusted Web Activity

Rather than rebuilding Wearly in Kotlin/Java, the Android app uses Google's TWA technology — a native Android shell loading the Vercel-hosted PWA full-screen. No browser chrome, native launcher icon, full camera + microphone + speech access.

Build Process

Built programmatically using @bubblewrap/core Node.js API (CLI has Node 26 incompatibility with readline). Key excerpt:

const manifest = new TwaManifest({
  packageId: 'com.wearly.app',
  host: 'wearly-dusky.vercel.app',
  name: 'Wearly AI',
  themeColor: '#2C4A1E',
  backgroundColor: '#080f06',
  appVersionName: '1.0.0',
  minSdkVersion: 21,  // Android 5.0+
});
const gradleWrapper = new GradleWrapper(process, androidSdkTools);
await gradleWrapper.assembleRelease();
await androidSdkTools.apksigner(keystore, pass, alias, keyPass, aligned, signed);

8. Privacy & Sustainability Design

Privacy-First Architecture

  • No account required — works without signing up for anything
  • No database — all wardrobe data lives in browser localStorage
  • No telemetry — no analytics, no tracking, no usage data collected
  • Local AI default — when Ollama is running, zero data leaves the device
  • Images stay local — photos stored as base64 in localStorage, never uploaded
  • Transparent fallback — UI shows whether Ollama or Groq is active

Sustainability Mission

Fashion is the second most polluting industry globally. Wearly's sustainability features are core to the product's purpose:

  • Wear Count + Cost-per-Wear — seeing "£45 jacket at £0.50/wear" changes buying behaviour
  • CO₂ Estimation — synthetic fast-fashion top ~15 kg CO₂e; natural fibre quality piece ~5–8 kg CO₂e; pre-owned ~1–2 kg CO₂e
  • 60-Day Idle Detection — identifies unworn items; suggests donate, sell, or re-discover
  • Circular Economy — one-click marketplace listing generation for resale platforms
  • Wardrobe Score (0–100) — holistic metric combining wear distribution, category balance, colour versatility, eco profile

11. What's Next

Short-term (1–3 months)

  • Gemma 4 fine-tuning on fashion-specific image datasets for improved colour and material accuracy
  • Outfit history — track which outfits worn on which days; Gemma 4 finds rotation patterns
  • Body proportion analysis — Gemma 4 suggests cuts and silhouettes based on measurements

Medium-term (3–6 months)

  • Offline Gemma 4 on Android — bundle quantized model using MediaPipe for fully offline analysis
  • Resale platform integration — one-tap listing push to Carousell, Depop, Poshmark via API
  • Seasonal rotation reminders — proactive Gemma 4 suggestions as seasons change

Long-term Vision

  • Personal style graph — Gemma 4 builds a semantic model of personal style preferences over time
  • AR try-on — overlay outfit suggestions on camera feed using WebXR
  • Privacy-preserving community — style sharing without exposing individual wardrobes

Closing Note

Wearly demonstrates that a genuinely useful, privacy-respecting consumer AI product can be built and shipped end-to-end using open models — no proprietary APIs, no data harvesting, no subscriptions required.

Gemma 4 is not just a component in Wearly — it is the product. Its vision capabilities make cataloguing effortless, its language capabilities make styling conversational, and its code generation capabilities make the product self-improving. Without Gemma 4, Wearly is a spreadsheet. With it, it's a personal stylist.
