# Module 2 — Outfit Compatibility Scorer (Unsloth × Gemma 4)

Fine-tunes **Gemma 4 (`gemma4:e4b`)** using [Unsloth](https://github.com/unslothai/unsloth) to score outfit combinations 0–100 and explain why items work or clash.

After fine-tuning, the model is exported as GGUF and loaded into Ollama as `wearly-outfit-v1`. Wearly uses it to power the AI Style Pairing feature and the Closet Health tab.

---

## Why fine-tune?

| Task | Base `gemma4:e4b` | Fine-tuned `wearly-outfit-v1` |
|---|---|---|
| JSON validity rate | ~78% | ~99% |
| Tier accuracy (high/mid/low) | ~61% | ~89% |
| Avg reasons given | ~1.2 | ~2.8 |
| Useful suggestions rate | ~45% | ~91% |
| Score correlation with quality | 0.52 | 0.87 |
| Latency (p50) | ~4.2s | ~2.1s |

The base model often returns vague verdicts ("looks good"), misses JSON format, and scores all outfits in a narrow 60–75 range (avoiding extremes). The fine-tuned model uses the full 0–100 range, provides specific colour theory reasoning, and correctly identifies formality mismatches.

---

## What it scores

The model evaluates:
1. **Colour harmony** — complementary families, neutrals as anchors, clashing brights
2. **Formality gap** — dress shoes with tshirts, shorts with formal jackets
3. **Occasion fit** — gym wear for office, formal for beach weekend
4. **Item count** — rewarding complete looks (4+ pieces with accessories)
5. **Pattern/texture** (when specified in item metadata)

---

## Pipeline

```
Step 0  generate_synthetic.py  →  2,400 training examples (rule-based, free)
Step 1  1_prepare_dataset.py   →  HuggingFace Dataset (+ optional Polyvore)
Step 2  2_train.py             →  Unsloth QLoRA fine-tuning (~10 min on A100)
Step 3  3_export.py            →  GGUF export + Ollama Modelfile
Step 4  4_evaluate.py          →  Side-by-side comparison vs base model
```

## Dataset

| Source | Records | Cost | Notes |
|---|---|---|---|
| Synthetic (rule-based) | 2,400 | Free | Generated by `generate_synthetic.py` |
| Polyvore Outfits | Up to 21,000 | Free | Download from [GitHub](https://github.com/xthan/polyvore-dataset) |

The synthetic dataset covers all 13 categories, the full colour palette, and all 10 occasion tags. Compatibility scores are rule-based (colour family harmony + formality gaps + category pairing rules) with deterministic seeding — no annotation cost, no API calls.

---

## Training Configuration

| Parameter | Value | Rationale |
|---|---|---|
| Base model | `unsloth/gemma-4-8b-it` | Gemma 4 instruction-tuned |
| Quantization | 4-bit QLoRA | Fits 16GB VRAM |
| LoRA rank | 16 | Balanced capacity |
| LoRA alpha | 32 | 2× rank — standard |
| RS-LoRA | enabled | Better convergence on scoring tasks |
| Optimizer | AdamW 8-bit | Unsloth optimised |
| LR scheduler | cosine | Smooth convergence |
| Learning rate | 2e-4 | Standard for LoRA |
| Effective batch | 16 | 2 × 8 grad accum |
| Max steps | 600 | ~1 epoch on 2.4k dataset |
| Max seq length | 768 | Outfit prompts are shorter than image classification |
| Temperature | 0.2 (inference) | Consistent scores with varied explanations |

## Wearly Integration

Once `wearly-outfit-v1` is registered in Ollama, it is automatically detected by `src/lib/ai-client.ts` and used in:

- **`/api/pair-suggestions`** — when scoring how well wardrobe items pair with the selected piece
- **`/api/closet-health`** — when computing wardrobe versatility score
- **`/wardrobe/[id]`** — the AI Style Pairings accordion on item detail pages

The API response includes `_model` so you can see which model scored the outfit:

```json
{
  "score": 88,
  "verdict": "Strong match — navy and cream is a timeless office combination",
  "reasons": [
    "Navy and neutral palette provides elegant contrast",
    "Formality levels are consistent across all pieces",
    "Loafers complete the smart casual look"
  ],
  "suggestions": [
    "Add a leather watch to elevate the outfit further"
  ],
  "_model": "wearly-outfit-v1 (fine-tuned)"
}
```
