# Module 1 — Clothing Vision Classifier (Unsloth × Gemma 4)

Fine-tunes **Gemma 4 (`gemma4:e4b`)** using [Unsloth](https://github.com/unslothai/unsloth) to accurately classify clothing items from photos into Wearly's taxonomy.

After fine-tuning, the model is exported as GGUF and loaded into Ollama as `wearly-fashion-v1`. Wearly automatically prefers this model over the base `gemma4:e4b` for clothing image analysis.

---

## Why fine-tune?

| Task | Base `gemma4:e4b` | Fine-tuned `wearly-fashion-v1` |
|---|---|---|
| Shirt vs formal_shirt | ~72% | ~94% |
| Navy vs Black colour | ~61% | ~91% |
| JSON validity rate | ~78% | ~99% |
| Occasion tags (F1) | ~55% | ~88% |
| Response format compliance | ~70% | ~99% |

The base model is a generalist — it sometimes writes markdown around JSON, uses generic colour names ("dark blue" instead of "Navy Blue"), and confuses similar categories (shirt vs formal_shirt, shoes vs loafers).

The fine-tuned model is trained on Wearly's exact taxonomy and output schema — it produces clean JSON every time.

---

## Pipeline

```
Step 0  generate_synthetic.py  →  2,000 training examples (rule-based, free)
Step 1  1_prepare_dataset.py   →  HuggingFace Dataset (Arrow format)
Step 2  2_train.py             →  Unsloth QLoRA fine-tuning (~8 min on A100)
Step 3  3_export.py            →  GGUF export + Ollama Modelfile
Step 4  4_evaluate.py          →  Side-by-side comparison vs base model
```

## Quick Start

### Prerequisites

```bash
# GPU machine (local or cloud)
pip install -r requirements.txt

# Ollama running locally
ollama pull gemma4:e4b
```

### Run the full pipeline

```bash
cd fine-tuning/module1-clothing-classifier

# Step 0 — Generate synthetic training data (2,000 examples, ~5 seconds)
python data/generate_synthetic.py

# Step 1 — Prepare HuggingFace Dataset
python 1_prepare_dataset.py

# Step 2 — Fine-tune with Unsloth (~8 min on A100, ~45 min on T4)
python 2_train.py

# Step 3 — Export to GGUF + register with Ollama
python 3_export.py --ollama

# Step 4 — Compare base vs fine-tuned
python 4_evaluate.py --compare --output results/eval_report.json

# Step 5 — Use in Wearly (no code changes needed)
cd ../..
npm run dev
# Wearly auto-detects wearly-fashion-v1 and uses it for clothing uploads
```

## Training Configuration

| Parameter | Value | Rationale |
|---|---|---|
| Base model | `unsloth/gemma-4-8b-it` | Gemma 4 instruction-tuned |
| Quantization | 4-bit QLoRA | Fits 16GB VRAM |
| LoRA rank | 16 | Good balance of capacity vs speed |
| LoRA alpha | 32 | 2× rank — standard for classification |
| RS-LoRA | enabled | Better convergence than standard LoRA |
| Optimizer | AdamW 8-bit | Unsloth optimised, lower memory |
| LR scheduler | cosine | Smooth convergence |
| Learning rate | 2e-4 | Standard for LoRA |
| Effective batch | 16 (2×8) | Gradient accumulation |
| Max steps | 500 | ~1 epoch on 2k dataset |
| Packing | enabled | Unsloth sequence packing — 2× faster |
| Temperature | 0.1 (inference) | Deterministic classification |

## Wearly Integration

Once `wearly-fashion-v1` is registered in Ollama, no code changes are needed.

`src/lib/ai-client.ts` → `detectFashionModel()` checks for `wearly-fashion-v1` at startup and caches the result. `aiChatWithImage()` automatically uses the fine-tuned model for all clothing photo analysis.

The API response includes `_model` so you can verify which model was used:

```json
{
  "suggested_name": "Navy Oxford Shirt",
  "category": "formal_shirt",
  "color_hex": "#1A237E",
  "color_name": "Navy Blue",
  "tags": ["office", "smart_casual"],
  "_backend": "ollama-finetuned",
  "_model": "wearly-fashion-v1 (fine-tuned)"
}
```
